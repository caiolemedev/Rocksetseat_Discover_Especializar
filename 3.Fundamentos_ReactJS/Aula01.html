Boas vindas as módulo de React!

Introdução

Uma biblioteca JS para criar interfaces de usuário. É chamado de biblioteca pois não tem uma opinião forte como um framework. Com ele podemos criar interfaces para web, mobile ou até desktop.

Criando Projeto

Acessamos a pasta do projeto pelo terminal, podemos usar o create react app ou vite. Usaremos o vite pois é muito mais performático e traz menos arquivos, somente o necessário, tornando a instalação mais simples e ocupando menos espaço. Rodamos o comando npm create vite@latest nomeDoProjeto --template React no terminal dentro da pasta do projeto.

Executando Projeto

O vite não traz a pasta node modules necessários para rodar o projeto, portanto precisamos entrar na pasta do projeto, reactapp e rodar o npm install para que as dependências sejam instaladas. Podemos na sequência rodar o projeto via npm run dev e a página será exibida em um localhost padrão.

Estrutura React

Ao entrarmos no arquivo index.html criado pelo vite podemos ver que existe apenas uma div root e um script rodando que vem da pasta src. Abrindo o conteúdo do main.jsx podemos ver diversas importações no arquivo, como ReactDOM, e renderiza na tela um componente de nome App dentro da div. Para importar componentes no React não precisamos colocar extensão. Dentro do arquivo App.jsx podemos encontrar todo o conteúdo renderizado.

JSX

O React utiliza a sintax chamada jsx, é uma maneira de escrever em React para poder criar as interfaces de forma declarativa. Nada mais é que uma função que retorna um html, mas é escrita em JS. Ele também permite trabalhar com tags muito similares ao html.

Estrutura Pastas e Arquivos

Entendendo os arquivos e pastas. Começando pelo index.html que o arquivo que inicializa o projeto, o package que organiza todas as dependências necessárias para o projeto e gitignore que não irá para o github pois podem ser baixadas facilmente pelo npm ou yarn. Pasta node_modules possui todas as dependências necessárias para rodar o projeto, organizadas em sub pastas. Pasta src terá os arquivos de renderização como o main.jsx que irá renderizar o conteúdo e o App.jsx que retorna o conteúdo html do site, bem como outros arquivos de suporte e CSS caso necessário. Aproveitamos para organizar as páginas da aplicação dentro de uma pasta, remover arquivos desnecessários e editar alguns caminhos.

Fragment

No JSX devemos retornar apenas 1 elemento html, podemos fazer via fragment que é basicamente um elemento nulo sem tag nenhuma. Ele é composto por uma abertura e fechamento sem nome. Ou podmeos usar uma div que da na mesma.

Importando arquivos via CSS

Para fazer a importação do CSS no React criamos os arquivos que vamos usar na pasta style e para utiliza-los na nossa aplicação usamos import e o caminho do arquivo.

Separando CSS

Utilizando a estrutura de pastas podemos separar cada uma das páginas da nossa aplicação e seu estilo, criando uma pasta por página contendo o arquivo index.jsx e o arquivo style.css. Não há necessidade de atualizar o nome da importação no main.jsx somente o caminho pois ele já irá procurar o arquivo padrão index. Assim realizamos a importação do estilo direto no index e refatoramos o código para que o export esteja antes do function.

Estilizando Página CSS

Para estilizar a página começamos colocando uma classe na div via className (parâmetros do react). Na sequência configuramos o CSS normalmente como um arquivo normal, ajustando a div e seus componentes com as características que desejamos.

Importando Fonte

Para importar fontes usamos o index html do site adicionando o link das fontes do google e no CSS global podemos utilizar a fonte importada para padronizar os textos da nossa página.

Componentes

O React é baseado em componentes para facilizar a replicação e reutilização de códigos. Criamos uma pasta separada chamada components que terá a mesma organização da pasta pages para os componentes. Criamos um componente card que foi importado para a página home e usado no formato de componente do React. Além disso podemos estilizar rapidamente o componente via CSS.

Propriedades

Para alterar as propriedades dos componentes precisamos primeiro passa-las no arquivo onde chamamos os componentes, texto passamos com aspas e número passamos com chaves. Para utiliza-las no component usamos o parâmetro props e acessamos cada uma das propriedades que setamos na chamada. Podemos simplificar a chamada do parâmetro props no componente usando chaves.
