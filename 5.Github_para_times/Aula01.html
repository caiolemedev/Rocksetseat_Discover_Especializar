Github para times

O que é o Github para times?

Podemos usar o Github para nos auxiliar no trabalho em time.

Criando um repositório

Criando um repositório teste podemos em confgurações adicionarmos os colaboradores do projeto e eles terão acesso ao mesmo repositório. Criamos um arquivo index.html e colocamos algumas informações básicas para termos um conteúdo no mesmo.

Branches

Na sequência, clonamos o repositório para o repositório local e abrimos o VS code local para editar os arquivos. Podemos editar o projeto em nossa branch main e alterarmos o projeto atual (podemos verificar no terminal ou no github desktop). Podemos usar o comando git checkout -b nomeDaNovaBranch e será criada a nova branch e irá nos posicionar nelas, para trocar depois de criada não precisamos do -b. Criamos mais um arquivo e realizamos algumas alterações na aplicação fazendo um commit na sequência. O log de commits é apresentado por branch, o que não estiver na main não aparece. No github podemos selecionar qual branch gostaríamos de trabalhar. Podemos usar branchs alternativas quando a main está no github pages e não queremos quebrar a página.

Merge

Para colocarmos o conteúdo de uma branch na main, colocamos o terminal dentro da main e usamos o comando git merge NomeDaBranchQueQueremosInserir. Assim todas as alterações da branch alteranativa serão inseridas na main. A branch não deixa de existir, permanece no mesmo status que deixamos.

Apagar uma branch

Quando não precisamos mais de uma branch e queremos apagar seu conteúdo, usamos o comando git brach -D branchASerApagada. Não podemos estar dentro da branch para apaga-la. A branch permanece no github e podemos deletar diretamente no site ou usando o comando git push origin --delete branchASerApagada no terminal.

Atualizando branchs

Quando temos a criação de outras branchs que não em nossa máquina local mas estão no github já podemos atualizar o status usando o comando git fetch no terminal ele atualizará na máquina local, mas ele não mostra na lista de branchs. Somente após fazermos um checkout para a branch atualizada é que a branch irá aparecer em nosso terminal. Se deletarmos a branch no github podemos usar o comando git fetch -p para atualizar com a deleções feitas no repositório, mas ainda precisamos usar o git branch -D branchASerApagada para deletar os arquivos também da nossa máquina.
