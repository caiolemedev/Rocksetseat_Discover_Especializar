Parâmetros nas requisições

O que são parâmetros nas requisições

Podemos programar nossa aplicação para receber parâmetros via url por exemplo: localhost:3000/?name=caio , ou no próprio body da requisição, como fizemos no método anterior via POST e PUT. Parâmetros são informações que passamos para o back-end.

Body Params

Podemos enviar os parâmetros de forma escondida vida body da requisição usando um arquivo json. Podemos receber o body via POST, PUT e PATCH. É possível realizar o desmembramento do body em variáveis para podemos trablhar com cada uma delas individualmente.

Route Params

Podemos usar também a própria rota para enviarmos informações. Dessa forma na rota do servidor devemos criar a variável que desejamos usar e ela receberá qualquer valor que for passado na rota.

Query Params

Além dos anteriores, é possível receber informações via query. Neste modelo mantemos a rota inalterada e usamos o ? para idenficiar a variável e o valor na url. Separamos as variáveis com &. Será recebido um arquivo json com as informações e podemos acessar valores específicos adicionando o parâmetro específico desejado. Exemplo: localhost:3000/about/user?nome=caio&cidade=São Paulo&id=974328.
